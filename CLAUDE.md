## Project Overview

**Chengyu Wordle** is a Wordle clone for Chinese idioms (成语/chéngyǔ). Players guess a 4-character idiom in 6 tries. Built with **Preact + Vite**, deployed as a static site. Each day has a fixed idiom (daily puzzle), but players can also play any past puzzle via idiom ID or random selection.

## Key Commands

```bash
npm i              # Install dependencies
npm run dev        # Start dev server on port 3030
npm run build      # Production build
npm run preview    # Preview production build locally
npm run data       # Regenerate game data from raw data/ into game-data/
npm test           # Run unit tests (vitest, watch mode)
npm run e2e-test   # Run E2E tests (Playwright, auto-starts dev server)
```

## Architecture

### Tech Stack
- **Preact** 10.x (React-compatible, lighter alternative) with JSX
- **Vite** 2.x as bundler/dev server
- **i18next** + react-i18next for localization (en, zh-CN)
- **howler.js** for sound effects
- **pinyin-pro** for Chinese pinyin conversion
- **vitest** for unit tests, **Playwright** for E2E tests

### Code Structure

```
src/
  main.jsx              # Entry point: initializes i18n, renders <App>
  app.jsx               # Main game logic (1500+ lines) — board, keyboard, modals, state
  constants.js          # localStorage key prefix ('cywd-')
  index.css             # Global styles
  components/           # Preact components (Tile, icons, etc.)
  utils/                # Helpers: compareWords, LS (localStorage), copy, alert, etc.

data/                   # Raw source data (idioms.json, THUOCL frequency list)
game-data/              # Processed game data (generated by `npm run data`)
  all-idioms.txt        # All valid idioms (one per line) — used for input validation
  game-idioms.csv       # Game idioms with unique IDs — daily puzzle pool
i18n/                   # Translation files (en.json, zh-CN.json)
test/                   # Unit tests (vitest)
e2e-test/               # E2E tests (Playwright)
```

### Game Flow

1. **Daily puzzle**: `getTodayGame()` selects an idiom based on days since 2022-01-27, cycling through the game-idioms list.
2. **Board state**: 6 rows × 4 characters. State managed in `board` array, persisted to localStorage.
3. **Input**: Players type pinyin letters (a-z mapped to Chinese characters via pinyin-pro), or click on-screen keyboard keys.
4. **Validation**: Submitted idioms must exist in `all-idioms.txt`.
5. **Scoring**: `compareWords()` returns 🟩 (correct position), 🟧 (wrong position), ⬜ (absent).
6. **Keyboard keys**: Dynamically generated per puzzle from related idioms to ensure enough valid answers exist.
7. **Hash routing**: `#<idiom-id>` loads a specific puzzle; `#/` loads today's game.

### Key Constants
- `MAX_STEPS = 6` (max guesses)
- `MAX_LETTERS = 4` (4 characters per idiom)
- `KEY_PREFIX = 'cywd-'` (localStorage key prefix)
- Hard mode exists (40 keys, 10 min idioms vs 20 keys, 6 min)

### Data Pipeline
`npm run data` (`scripts/gen-data.mjs`):
1. Reads `data/idioms.json` (all idioms) + `data/THUOCL_chengyu.txt` (high-frequency list)
2. Filters to 4-character idioms
3. Generates `game-data/game-idioms.csv` with hash-based IDs
4. Generates `game-data/all-idioms.txt` (union of all idioms for validation)

## Testing

- **Unit tests**: `test/compareWords.test.js` — tests the tile comparison logic. Run with `npm test`.
- **E2E tests**: `e2e-test/` — Playwright tests for app loading, gameplay, and localization. Run with `npm run e2e-test`. Auto-starts dev server on port 3030.

## Notes

- The main `app.jsx` is a single large component (~1500 lines) containing all game logic. This is intentional — the project is a small game, not a large app.
- Game state is persisted to localStorage with the `cywd-` prefix. Helper functions `window.clearGames()` and `window.allGames()` are exposed for debugging.
- The answer and hints are exposed in browser console (`window.ANSWER`, `window.HINTS`) as easter eggs.
- Definition data is fetched from an external API (`baidu-hanyu-idiom.cheeaun.workers.dev`) at runtime.
